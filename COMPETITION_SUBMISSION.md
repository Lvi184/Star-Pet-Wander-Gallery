## 1. Demo 简介

**是什么：** 一款基于 AI Town + Generative Agents 设计思想的自主生命体养成游戏，玩家作为陪伴者和见证者，观察灵兽在山海灵境世界中自由探索、社交互动、成长进化。

**面向谁：** 喜欢治愈系宠物养成、探索收集、叙事体验的游戏玩家，特别是对 AI 驱动的自主角色行为感兴趣的用户。

**主要功能：**

- 🎮 **在线操控模式**：玩家可直接操控灵兽在地图上移动探索，点击地图即可移动角色
- 🤖 **离线AI接管**：玩家下线后，AI 自动接管灵兽，继续在世界中自主探索、社交、休息
- 📜 **探索记录系统**：右侧状态栏实时记录灵兽的探索历程，包括发现事件、采集物品等
- 💬 **多宠物社交对话**：灵兽之间可相互邀请对话，对话内容由 LLM 实时生成
- 🔄 **控制切换**：支持一键切换玩家控制/AI 控制模式，随时打断或恢复自主行为
- 🗺️ **山海灵境地图**：包含青丘秘境、昆仑之墟、东海深渊等多个探索区域，每个区域有专属事件

## 2. Demo 创作思路

**灵感来源：**

传统电子宠物游戏需要玩家持续喂养照顾，容易产生疲劳感。我希望创造一种新的宠物养成模式——灵兽有自己的性格和意志，玩家是陪伴者而非照顾者。参考了 Stanford 的 Generative Agents 论文和 AI Town 项目，希望实现"离线后宠物依然在生活"的沉浸式体验。

**想解决的问题：**

1. 传统宠物养成的"喂养疲劳"问题——玩家不需要持续照顾
2. 角色行为的"机械感"问题——通过 AI 驱动让角色行为更加自然、有个性
3. 游戏体验的"碎片化"问题——即使离线，灵兽也在积累故事

**为什么做这个方向：**

选择"自主生命体"作为核心方向，是因为这是当前 AI 游戏领域最前沿的探索方向之一。将 LangGraph、多智能体协作、记忆机制等技术与游戏结合，可以创造出传统游戏无法实现的动态叙事体验。

## 3. Demo 体验地址

**代码仓库：** https://github.com/Lvi184/Star-Pet-Wander-Gallery

**开发环境体验：** http://localhost:3003/

**操作指南：**
1. 打开页面后，左侧会显示星宠名册（包含"哈基咪"等角色）
2. 点击角色卡片即可选中该宠物
3. 点击地图任意位置，选中的宠物会自动移动过去（A* 寻路）
4. 点击角色卡片上的 🤖/🎮 按钮，可切换 AI 控制/玩家控制模式
5. 右侧"探索记录"面板会实时显示宠物的探索历程

**核心体验点：**
- 选择"哈基咪"角色，点击地图移动 → 体验在线操控
- 点击 🤖 切换 AI 模式 → 观察宠物自主探索
- 查看右侧探索记录 → 见证宠物的冒险故事

**本地运行方式：**
```bash
# 克隆仓库
git clone https://github.com/Lvi184/Star-Pet-Wander-Gallery.git
cd Star-Pet-Wander-Gallery

# 前端
cd frontend
npm install
npm run dev

# 后端（可选）
cd backend
pip install -r requirements.txt
python main.py
```

## 4. TRAE 实践过程

### 开发流程概览

项目采用前后端分离架构，核心开发流程如下：

1. **需求分析与架构设计** → 确定自主生命体 + AI Town 设计方向
2. **前端基础框架搭建** → React + TypeScript + Vite + Pixi.js
3. **游戏引擎开发** → 实现地图渲染、角色移动、碰撞检测
4. **AI 行为引擎开发** → 基于 LangGraph 的多智能体决策系统
5. **对话系统实现** → LLM 驱动的 NPC 社交对话
6. **探索记录系统** → 事件记录与实时展示
7. **控制切换功能** → 在线/离线无缝接管

### 关键技术实现

**AI 行为决策流程（参考 Generative Agents）：**

```
Observation → Memory → Reflection → Planning → Action
```

- **Memory（记忆系统）**：实现 SQLite 持久化存储，支持时间加权检索、短期/长期记忆自动转换
- **Planning（规划系统）**：基于日程和当前状态生成下一步行动
- **Reflection（反思系统）**：定期生成反思摘要，影响后续行为决策

**对话状态机：**
```
invited → walkingOver → participating → leave
```

灵兽之间通过完整的对话生命周期进行社交互动，对话内容由 DeepSeek API 生成。

## 项目特色

1. **自主生命体设计**：灵兽有独立性格，不会完全听从玩家，体现"陪伴者而非照顾者"的理念
2. **离线自主行为**：玩家离线后 AI 继续驱动角色行动，形成完整的生活叙事闭环
3. **命运与死亡机制**：包含每日命运等级、受伤、迷路、死亡等风险元素，死亡是叙事节点而非失败
4. **天道回溯功能**：支持回档后世界线变化，观察不同人生可能性
5. **多宠物社交**：参考 CAMEL 的多智能体协作逻辑，灵兽之间可自由对话
