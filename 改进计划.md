## Phase 1: 接入真实 LangGraph + DeepSeek API

### 目标

将当前纯 Python 状态机替换为真实 LangGraph，实现完整的 Agent 工作流

### 关键改进

#### 1. Graph 重构

参考 LangGraph 官方 API 和经验教训（Experience ID: 1162202）：

- 使用 `langgraph.StateGraph` 替代纯 Python 流程
- 定义 `PetGraphState` TypedDict，每个字段使用 `Annotated[..., operator.add]` 作为 reducer
- 使用 `add_conditional_edges` 实现决策分支（如能量不足→休息，精力充足→探险）

**注意事项**：
- `add_conditional_edges` 的 routing_func 返回值必须与 ends 的 key 类型一致
- ends 必须是 `dict: {<route_key>: <node_name_or_END>}`，且 node_name 必须是已注册的字符串
- 禁止把 Python 函数对象当作 target 传入

#### 2. DeepSeek API 接入

- 在 `.env` 中配置 `DEEPSEEK_API_KEY`
- 使用 `langchain_openai.ChatOpenAI` 兼容模式调用 DeepSeek API
- 添加 API 超时重试机制（指数退避）

#### 3. Memory 持久化

- 将 `memory.py` 的内存存储改为 SQLite 持久化
- 添加记忆检索优化（时间加权 + 重要度排序）

### 文件改动

| 文件 | 操作 | 描述 |
|------|------|------|
| `backend/ai/graph.py` | 重构 | 替换为真实 LangGraph StateGraph |
| `backend/ai/memory.py` | 修改 | 添加 SQLite 数据库持久化 |
| `backend/config.py` | 修改 | 添加 API 重试配置 |
| `backend/models/memory.py` | 新增 | 记忆数据模型 |
| `backend/.env` | 新增 | DeepSeek API Key 配置 |

### 执行步骤

```bash
# 1. 安装依赖
pip install langgraph langchain langchain-openai

# 2. 配置 API Key
echo "DEEPSEEK_API_KEY=your_key" > backend/.env

# 3. 运行测试
cd backend
python -c "import asyncio; from ai.graph import run_pet_behavior_graph; result = asyncio.run(run_pet_behavior_graph('test', {})); print(result)"
```

## Phase 3: 添加学习与经验系统（Voyager）

### 目标

参考 Voyager，实现宠物从失败中学习的能力

### 关键改进

#### 1. 经验模块

- 记录每次探险的成功/失败经验
- 例如："森林危险→以后绕路"

#### 2. 学习 Agent

- 分析失败原因，生成改进策略
- 将经验写入长期记忆，影响后续决策

#### 3. 决策优化

- Planner Agent 在决策时考虑历史经验
- 宠物越探索越聪明，避开已知危险区域

### 参考框架与理论基础

#### 第一梯队（强烈建议研究）

**1. AI Town（Google + Stanford）**
- **项目地址**：`https://github.com/a16z-infra/ai-town`
- **核心思想**：玩家不是控制角色，而是观察者。NPC 有自己的记忆、日程，会互相聊天、形成故事
- **关键机制**：
  - **Memory**：长期记忆（"今天认识了一只猫"）+ 短期记忆（"刚刚捡到一块星星碎片"）
  - **Daily Schedule**：早晨探索 → 下午交朋友 → 晚上回家睡觉
  - **Reflection**：每天总结（今天最大的收获是什么？今天最开心的是？今天想再去哪里？）
- **参考价值**：⭐⭐⭐⭐⭐

**2. Generative Agents（Stanford University）**
- **项目地址**：`https://github.com/langchain-ai/generative-agents`
- **经典论文**：《Generative Agents: Interactive Simulacra of Human Behavior》
- **核心流程**：
  ```
  Observation → Memory → Reflection → Planning → Action
  ```
- **示例**：天气很好 → 昨天没去森林 → 今天适合探险 → 去森林 → 探索森林
- **参考价值**：⭐⭐⭐⭐⭐

**3. Voyager（NVIDIA）**
- **项目地址**：`https://github.com/MineDojo/Voyager`
- **核心思想**：思考→行动→失败→总结经验→再次行动，宠物越探索越聪明
- **示例**：森林危险 → 以后绕路 → 写进经验
- **参考价值**：⭐⭐⭐⭐⭐

#### 第二梯队（非常值得参考）

### 经验教训

参考 Experience ID: 1162202 的关键教训：

1. **ImportError 处理**：优先利用报错中的 "Did you mean" 提示，按"包名→模块→导出符号"逐层对齐
2. **LangGraph reducer**：channels/binop reducer 必须是二参函数，使用 `operator.add`
3. **add_conditional_edges**：routing_func 返回值类型与 ends 的 key 类型必须一致
4. **模块命名**：避免使用 `utils` 等常见第三方包名，防止导入冲突
