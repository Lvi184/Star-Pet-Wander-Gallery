PRD 需求分析工具，自动生成标准产品需求文档。

**输入**: 用户描述的模糊产品需求或功能想法

**PRD 文档结构**

1. **需求概述**
   - 需求背景
   - 目标用户
   - 业务目标

2. **功能描述**
   - 功能清单
   - 功能优先级
   - 核心流程

3. **交互逻辑**
   - 用户路径
   - 页面流程图
   - 交互细节

4. **验收标准**
   - 功能验证点
   - 性能指标
   - 兼容性要求

5. **非功能需求**
   - 安全性
   - 可扩展性
   - 可维护性

**Steps**

1. **需求收集**
   - 理解用户需求
   - 确认需求范围
   - 收集关键信息

2. **文档生成**
   - 按照标准结构编写
   - 确保内容完整
   - 保持逻辑清晰

3. **审核确认**
   - 检查文档完整性
   - 确认需求理解正确
   - 输出最终文档

**输出**

- 标准 PRD 文档
- 功能清单表格
- 用户路径图说明

**使用示例**

```
@prd-analyzer 分析宠物社交功能需求
@prd-analyzer 生成探险系统PRD文档
@prd-analyzer 梳理复赛产品功能清单
```
