**当前状态：**
- 项目已集成 Phaser.js 游戏引擎
- 已有基本的玩家控制和场景切换功能
- 资源目录 `frontend/public/assets/game/` 已存在
- 参考资源位于 `E:\兴趣\全民宠物（星宠漫游馆）\资源\界面` 和 `E:\兴趣\全民宠物（星宠漫游馆）\资源\参考游戏`

## Decisions

### 1. 资源复用策略

**决策：** 将参考项目的资源文件复制到项目的 `public/assets/game/` 目录，按场景分类组织。

**理由：**
- Phaser.js 需要从静态资源路径加载地图和精灵图
- 按场景分类便于管理和维护
- 避免修改参考项目的原始文件

**目录结构：**
```
public/assets/game/
├── forest/          # 青丘森林场景资源
│   ├── tilemap.json
│   ├── rpg-tileset.png
│   └── sprites/
└── town/            # 星宠小镇场景资源
    ├── map.json
    ├── tilesets/    # 8个瓦片集
    └── sprites/     # NPC 精灵图
```

### 2. 场景架构设计

**决策：** 使用 Phaser 的 Scene 系统，为每个场景创建独立的 Scene 类。

**理由：**
- Phaser 的 Scene 系统提供了完整的生命周期管理
- 每个场景可以独立加载资源和管理状态
- 便于场景切换和状态保持

**场景类：**
- `ForestScene`: 青丘森林场景
- `TownScene`: 星宠小镇场景

### 3. 场景切换机制

**决策：** 通过 React refs 暴露 Phaser 场景切换方法，实现 UI 按钮控制。

**理由：**
- React 和 Phaser 是独立的渲染系统，需要通过 refs 通信
- useImperativeHandle 提供了类型安全的方法暴露
- 避免直接操作 DOM 元素

**实现方式：**
- GameEngine 组件使用 forwardRef 和 useImperativeHandle 暴露 switchScene 方法
- Explore 页面通过 gameRef.current.switchScene('TownScene') 触发切换

### 4. NPC 对话系统设计

**决策：** 使用 Phaser 的 Text 和 Graphics 对象实现对话框，靠近检测使用距离计算。

**理由：**
- 纯 Phaser 实现避免 React 和 Phaser 的渲染冲突
- 距离计算简单高效，适合小型场景
- Text 对象支持多行文本和样式自定义

**实现方式：**
- 在场景 update 中计算玩家与 NPC 的距离
- 距离小于阈值时显示对话提示
- 按空格键创建对话框显示对话内容

### 5. 碰撞检测实现

**决策：** 使用 Phaser Arcade Physics 的静态碰撞体，通过瓦片地图的碰撞层配置。

**理由：**
- Arcade Physics 轻量级，适合 2D 顶视角游戏
- 瓦片地图的碰撞层提供了精确的碰撞区域定义
- 无需自定义碰撞检测逻辑

**实现方式：**
- 在地图加载时读取 collision 层数据
- 创建静态碰撞体组
- 配置玩家与碰撞体组的碰撞检测

## Risks / Trade-offs

### 风险：资源路径不一致

**风险：** 参考项目的资源路径与项目实际路径不一致，导致加载失败。

**缓解措施：**
- 在资源复制时统一重命名和路径调整
- 在 Scene 中使用相对路径加载资源
- 添加资源加载失败的错误处理

### 风险：地图尺寸适配

**风险：** 参考地图的尺寸与游戏画布尺寸不匹配，导致显示不全或拉伸。

**缓解措施：**
- 使用 Phaser 的 Camera 系统实现地图滚动
- 设置合适的相机跟随玩家
- 调整游戏画布尺寸以适应地图比例

### 风险：精灵图帧配置错误

**风险：** 精灵图的帧大小和数量配置错误，导致动画播放异常。

**缓解措施：**
- 仔细检查参考项目的精灵图配置
- 在加载时输出调试信息验证帧配置
- 使用精灵图查看工具确认帧大小

### 风险：性能问题

**风险：** 复杂的瓦片地图和动画可能导致帧率下降。

**缓解措施：**
- 使用 Canvas 渲染模式而非 WebGL
- 限制同时渲染的精灵数量
- 避免不必要的 update 计算

### 风险：场景切换状态丢失

**风险：** 场景切换时玩家位置等状态丢失。

**缓解措施：**
- 在场景切换前保存玩家状态到全局变量
- 在场景初始化时恢复玩家状态
- 使用 Phaser 的 Scene 生命周期回调管理状态
